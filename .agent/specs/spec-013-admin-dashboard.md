## 详细设计

### 1. 管理后台前端架构
- Next.js 14 项目初始化
- Admin Layout + 菜单路由（侧边栏 7 项导航）
- UI 框架：深色玻璃极光风格（Apple Vision OS + 玻璃卡片）

### 2. 管理后台页面

| 页面 | 对应功能 |
|------|---------|
| 登录页 | JWT 认证 |
| 数据看板 Dashboard | KPI卡片 + Token/ROI/调用量图表 |
| 用户管理 | CRUD + 角色分配 + 启停（system_admin/admin/user 三级） |
| 权限管理 | 角色定义 + 权限映射 |
| 模型配置 | LLM 连接配置 + 参数调整（仅 system_admin） |
| 系统配置 | 全局参数设置（仅 system_admin） |
| 任务管理 | 全局任务监控 + 取消/重试 |
| 知识库管理 | 上传 + 列表 + 搜索 + 索引状态 |
| 审计日志 | 筛选 + 导出 |
| API 转换审核 | OpenAPI 导入 + 审批 |
| 站内信 | 收发/群发/全站发送 |
| 修改密码 | 个人设置页 |

- 响应式适配（移动浏览器兼容）

### 3. 密码管理

- **修改自己密码**：登录后所有角色均可通过后台「修改密码」页修改
- **系统管理员初始密码**（见 SPEC-003）：
  - 系统启动时自动生成随机密码，日志输出
  - 登录后头部横幅通知提示修改
  - 修改后 `password_changed=true`，横幅消失

### 4. 站内信系统

- MongoDB `notifications` 集合存储
- **发送类型**：
  - **互发（点对点）**：system_admin ↔ admin / admin ↔ user，无数量限制
  - **群发**：选择多个接收人，每日每发送人上限 50 条
  - **全站发送**：仅 system_admin，发送到通知列表页（不产生 N 条独立数据，避免数据膨胀）
- **展示**：后台头部铃铛图标 + 未读数红点；点击展开通知列表
- **状态**：未读 → 已读（点击查看后标记）
- **持久化**：TTL 90 天自动清理

### 5. 删除确认

- 所有删除操作（用户、知识库文档、任务、API 转换审核）弹出确认对话框
- 确认内容：「确定要删除 XXX 吗？此操作不可撤销。」
- 按钮：取消（默认焦点）/ 确认删除（红色警告按钮）
