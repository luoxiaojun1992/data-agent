# UI E2E 测试设计 — 错误状态与边界条件 (ERR)

> **SPEC-041** | Status: 设计中 | Date: 2026-07-09 | Phase: P8

## 1. 目标

定义 DataAgent 错误状态与边界条件的 E2E UI 测试用例规范。覆盖网络断开提示、API 500 错误处理、404 页面、空数据状态、加载状态 Skeleton、熔断器打开状态提示和浏览器后退按钮行为。

> **设计依据**: `docs/DataAgent-UI测试用例文档.md` §28
>
> **测试框架**: Playwright (TypeScript)
>
> **用例范围**: UI-196 ~ UI-202（共 7 个用例）

## 4. 测试用例

### UI-196: Err — 网络断开提示

- **优先级**: P0
- **前置条件**: 已登录
- **步骤**:
  1. 断开网络连接
  2. 尝试操作（发送消息、保存配置等）
- **预期结果**:
  - 显示网络错误提示（toast 或内联提示）
  - 操作按钮保持禁用直到网络恢复
  - 网络恢复后自动重试或提示用户重试
  - `data-testid`: `network-error-toast`

### UI-197: Err — API 500 错误处理

- **优先级**: P1
- **前置条件**: 模拟后端 500 错误
- **步骤**:
  1. 发送 Chat 消息
- **预期结果**:
  - 显示友好错误提示：「服务暂时不可用，请稍后重试」
  - 不显示技术栈堆信息
  - `data-testid`: `api-error-toast`

### UI-198: Err — 404 页面

- **优先级**: P1
- **前置条件**: 访问不存在的路由
- **步骤**:
  1. 输入 `/nonexistent` URL
- **预期结果**:
  - 显示 404 页面：「页面未找到」
  - 包含返回首页链接
  - `data-testid`: `page-404`, `page-404-home-link`

### UI-199: Err — 空数据状态

- **优先级**: P1
- **前置条件**: 任何列表页无数据
- **步骤**:
  1. 检查空状态 UI
- **预期结果**:
  - 显示空状态插图/图标
  - 显示引导文字（如「暂无数据」「还没有任务，点击创建第一个任务」）
  - 「创建」按钮（如适用）
  - `data-testid`: `{page}-empty-state`, `{page}-empty-action`

### UI-200: Err — 加载状态 Skeleton

- **优先级**: P1
- **前置条件**: 首次加载数据量大的页面
- **步骤**:
  1. 打开用户管理页（模拟慢网速）
- **预期结果**:
  - 显示 Skeleton loading（灰色占位块）
  - 数据加载完成后替换为实际内容
  - `data-testid`: `{page}-skeleton`

### UI-201: Err — 熔断器打开状态提示

- **优先级**: P1
- **前置条件**: 某 Skill 连续失败 5 次导致熔断器打开（SPEC-004 §6）
- **步骤**:
  1. 尝试触发该 Skill 的操作
- **预期结果**:
  - 返回 HTTP 503 或前端显示提示：「该服务暂时不可用，正在恢复中…」
  - 不继续尝试调用
  - 冷却 30s 后熔断器半开可探测
  - `data-testid`: `circuit-breaker-open-toast`
- **相关 Spec**: SPEC-004 §6

### UI-202: Err — 浏览器后退按钮行为

- **优先级**: P1
- **前置条件**: 已登录并浏览多个页面
- **步骤**:
  1. 从 Dashboard 导航到用户管理
  2. 点击浏览器后退按钮
- **预期结果**:
  - 返回到 Dashboard
  - 页面状态保留（如筛选条件、分页位置）
  - `data-testid`: N/A

## 5. 可行性分析

| 检查项 | 结论 |
|--------|------|
| 是否需要模拟后端 API | Yes — 各页面的错误响应 |
| 是否依赖第三方服务 | No |
| 是否需要特殊测试数据 | Yes — 空数据、500 错误、503 错误 mock 数据 |
| 是否需要新增 DB 集合 | No |
| 是否影响现有 API | No |
| 性能影响 | 无 |
| 实现复杂度 | Medium — 需要 mock 多种错误状态和边界条件 |
| 是否有无法实现的用例 | **待确认** — `UI-196` 网络断开：Playwright 的 `page.route()` 可以 abort 请求来模拟；`UI-200` Skeleton loading 需要控制 API 响应延迟；`UI-201` 熔断器状态需要 mock 特定的 HTTP 503 响应；`UI-202` 浏览器后退按钮：Playwright 支持 `page.goBack()` 但页面状态保留取决于前端框架实现 |
