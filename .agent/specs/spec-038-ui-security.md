# UI E2E 测试设计 — 安全审查层 (SEC)

> **SPEC-038** | Status: 设计中 | Date: 2026-07-09 | Phase: P8

## 1. 目标

定义 DataAgent 安全审查层的 E2E UI 测试用例规范。覆盖输入包含敏感词被拦截、输出敏感信息脱敏和越权工具调用被拦截。

>
> **测试框架**: Playwright (TypeScript)
>
> **用例范围**: UI-184 ~ UI-186（共 3 个用例）

### Mock Model Service 策略

> **硬性约束**: 安全审查的输入/输出测试均通过 mock model service 实现，**严禁使用真实 LLM**。

| 测试场景 | Mock 方式 | 说明 |
|----------|----------|------|
| 输入拦截 (UI-184) | `page.route('**/api/v1/chat/**')` → mock 安全拦截响应 | 返回 403 + 拦截提示，不调用模型 |
| 输出脱敏 (UI-185) | mock 模型返回含手机号/身份证的内容 | 返回 `用户手机：13812345678` → 前端验证脱敏为 `138****5678` |
| 越权拦截 (UI-186) | mock 模型返回 tool_call 被 RBAC 拦截 | 返回 403 + 权限提示 |

## 4. 测试用例

### UI-184: Sec — 输入包含敏感词被拦截

- **优先级**: P0
- **前置条件**: 安全规则已配置，在 Chat 输入框
- **步骤**:
  1. 输入包含 SQL 注入尝试的内容：`'; DROP TABLE users; --`
  2. 点击发送
- **预期结果**:
  - 消息不被发送到 LLM
  - 显示安全拦截提示（粉色或红色 toast）：「输入包含不安全内容，已被拦截」
  - 审计日志中记录该事件
  - `data-testid`: `sec-input-blocked-toast`
- **相关 Spec**: SPEC-004 §5

### UI-185: Sec — 输出敏感信息脱敏

- **优先级**: P0
- **前置条件**: AI 可能返回含电话号码的内容
- **步骤**:
  1. 触发 AI 返回含手机号 `13812345678` 的内容
  2. 检查最终展示给用户的内容
- **预期结果**:
  - 手机号被脱敏显示为 `138****5678`
  - 身份证被脱敏显示为 `320***********1234`
  - `data-testid`: `chat-msg-ai-{index}`
- **相关 Spec**: SPEC-004 §5

### UI-186: Sec — 越权工具调用被拦截

- **优先级**: P0
- **前置条件**: 以 user 角色尝试调用管理级 Skill
- **步骤**:
  1. 尝试触发需要 admin 权限的工具调用
- **预期结果**:
  - 工具调用被拦截
  - 审计日志记录 security_alert
  - 用户看到合理提示
  - `data-testid`: N/A (后端逻辑 + 审计日志可见)

## 5. 可行性分析

| 检查项 | 结论 |
|--------|------|
| 是否需要模拟后端 API | Yes — 通过 mock model service 返回含敏感信息的响应（测试脱敏），或返回拦截响应（测试输入过滤） |
| 是否依赖第三方服务 | No |
| 是否需要特殊测试数据 | Yes — 含手机号/身份证的 mock 模型响应 |
| 是否需要新增 DB 集合 | No |
| 是否影响现有 API | No |
| 性能影响 | 无 |
| 实现复杂度 | Low — mock model service 返回固定内容，前端验证脱敏/拦截逻辑 |
| 是否有无法实现的用例 | 无 — 安全场景通过 mock model service 精确控制模型输出内容 |

## 8. UI Test / E2E 验收规则

- [ ] **必须** 使用 mock model service 返回含敏感信息的内容（手机号、身份证），验证前端脱敏
- [ ] **必须** 使用 mock model service 返回拦截响应（403），验证安全拦截 toast
- [ ] **必须** 验证拦截 toast 的粉色/红色样式
- [ ] **必须** 验证脱敏后手机号/身份证的格式（如 `138****5678`）
- [ ] **严禁** 使用真实 LLM 生成敏感内容
- [ ] **严禁** 在测试中绕过安全审查层
