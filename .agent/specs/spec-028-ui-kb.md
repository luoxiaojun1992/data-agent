## 4. 测试用例

### UI-115: KB — 知识库管理页渲染

- **优先级**: P0
- **前置条件**: 以知识管理员或 system_admin 登录，点击「知识库管理」
- **步骤**:
  1. 检查页面
- **预期结果**:
  - Header：「知识库管理 · LLM 索引」
  - 「+ 上传文档」按钮（蓝紫渐变）
  - 「📦 批量上传」按钮
  - 蓝色信息条：「📌 文档上传后由 LLM 自动分析语义段落边界并拆分索引（复用异步 Agent 任务），索引数据与文档 ID 强绑定」
  - 文档卡片列表
  - `data-testid`: `kb-page-header`, `kb-upload-btn`, `kb-batch-upload-btn`, `kb-info-banner`
- **相关设计**: UI原型设计文档 §3.10

### UI-116: KB — 文档卡片渲染

- **优先级**: P0
- **前置条件**: 知识库中有文档
- **步骤**:
  1. 检查文档卡片
- **预期结果**:
  - 玻璃卡片背景 + 玻璃边框 + 内边距 20px 24px
  - 左侧文件图标（44x44px 圆角方，粉色背景）
  - 中间：文件名（15px Bold）+ 元数据（12px `#7A7A7A`：文件大小 + 分片数）
  - 右侧：索引状态 Pill + 标签 (tag)
  - 索引状态：
    - ✅ 已索引 ✓：绿色 Pill (`#34D399`)
    - 🔄 索引中 ⟳：琥珀色 Pill (`#FBBF24`) + 进度旋转动画
    - ❌ 索引失败：粉色 Pill (`#FB7185`)
  - `data-testid`: `kb-doc-card-{docId}`, `kb-doc-name`, `kb-doc-meta`, `kb-doc-status`, `kb-doc-tags`
- **相关设计**: UI原型设计文档 §3.10

### UI-117: KB — 上传单个文档

- **优先级**: P0
- **前置条件**: 知识库管理页
- **步骤**:
  1. 点击「+ 上传文档」
  2. 选择本地 PDF 文件 `Q2_财务报告.pdf` (2.4 MB)
  3. 确认上传
- **预期结果**:
  - 文件上传进度条显示
  - 上传完成后新文档卡片出现
  - 索引状态初始为「索引中 ⟳」（琥珀色 + 动画）
  - 异步 Agent 任务自动触发
  - `data-testid`: `kb-upload-modal`, `kb-upload-file-select`, `kb-upload-progress`

### UI-118: KB — 批量上传文档

- **优先级**: P0
- **前置条件**: 知识库管理页

## 5. 可行性分析

| 检查项 | 结论 |
|--------|------|
| 是否需要模拟后端 API | Yes — 文档上传/下载/搜索/删除 API、索引状态轮询 API |
| 是否依赖第三方服务 | Yes — Qdrant 向量索引（需 mock） |
| 是否需要特殊测试数据 | Yes — 多种格式的测试文件、索引状态 mock 数据 |
| 是否需要新增 DB 集合 | No |
| 是否影响现有 API | No |
| 性能影响 | 无 |
| 实现复杂度 | **High** — `UI-117/UI-118` 文件上传需要 Playwright `setInputFiles` + mock 上传进度；`UI-119` 拖拽上传需要模拟 `dragenter`/`dragover`/`drop` 事件；`UI-120` 索引状态需要 mock 轮询或 WebSocket 更新 |
| 是否有无法实现的用例 | **待确认** — `UI-119` 拖拽上传：Playwright 文件拖拽有限制，需用 `page.dispatchEvent` 模拟；`UI-120` 索引状态实时更新需要 mock 状态变更的 API 响应 |
