# UI E2E 测试设计 — IM 集成（飞书）(IM)

> **SPEC-034** | Status: 设计中 | Date: 2026-07-09 | Phase: P8

## 1. 目标

定义 DataAgent IM 集成（飞书）模块的 E2E UI 测试用例规范。**E2E 自动化测试仅覆盖 Web 绑定页** (UI-161, UI-162)，飞书客户端侧功能 (UI-163 ~ UI-166) 标记为人工测试。

> **设计依据**: `docs/DataAgent-UI测试用例文档.md` §21
>
> **测试框架**: Playwright (TypeScript)
>
> **用例范围**: UI-161 ~ UI-166（共 6 个用例，其中 4 个为人工测试）

## 4. 测试用例

### UI-161: IM — 飞书用户绑定页（E2E 自动化）

- **优先级**: P1
- **前置条件**: 飞书用户首次使用机器人（通过绑定 URL + 有效 Token 访问）
- **步骤**:
  1. 访问绑定 URL（含有效 Token）
  2. 检查 Web 绑定页
  3. 输入系统账号密码
  4. 点击「绑定」
- **预期结果**:
  - Web 绑定页显示 DataAgent Logo
  - 输入框：邮箱 + 密码
  - 绑定成功后显示「绑定成功」，提示返回飞书
  - `data-testid`: `im-bind-page`, `im-bind-email`, `im-bind-password`, `im-bind-submit`

### UI-162: IM — 绑定 Token 有效期（E2E 自动化）

- **优先级**: P1
- **前置条件**: 使用过期 Token 访问绑定 URL
- **步骤**:
  1. 访问绑定 URL（带已过期的 Token）
- **预期结果**:
  - 页面显示：「绑定链接已过期，请在飞书中重新发送 /帮助 获取新链接」
  - `data-testid`: `im-bind-expired`

### UI-163: IM — 飞书卡片消息格式化（人工测试）

- **优先级**: P1
- **前置条件**: 已绑定用户，在飞书中发送分析请求
- **步骤**:
  1. 在飞书群中 @机器人 「本月销售TOP10」
  2. 检查返回的卡片消息
- **预期结果**:
  - 消息以飞书卡片格式呈现
  - 包含：表格数据 + 关键指标 + 图表链接
  - 卡片美观易读
- **测试方式**: 👤 人工在飞书客户端验证

### UI-164: IM — 快捷指令（人工测试）

- **优先级**: P1
- **前置条件**: 已绑定用户
- **步骤**:
  1. 发送 `/分析 本月华东区销售额`
  2. 发送 `/查询 库存周转率`
  3. 发送 `/周报`
  4. 发送 `/帮助`
- **预期结果**:
  - `/分析`：触发数据分析，返回分析结果
  - `/查询`：触发快速查询，返回查询结果
  - `/周报`：生成本周经营分析周报
  - `/帮助`：返回可用指令列表和使用说明
- **测试方式**: 👤 人工在飞书客户端验证
- **相关 PRD**: PRD §F-24

### UI-165: IM — 异步任务完成飞书通知（人工测试）

- **优先级**: P1
- **前置条件**: 已绑定用户，有异步 Agent 任务
- **步骤**:
  1. 技术端触发 Agent 任务完成事件
  2. 检查飞书消息
- **预期结果**:
  - 用户收到飞书消息通知
  - 消息包含：任务名称 + 完成时间 + 耗时 + 查看结果链接
- **测试方式**: 👤 人工在飞书客户端验证

### UI-166: IM — 未绑定用户引导（人工测试）

- **优先级**: P1
- **前置条件**: 飞书用户未绑定系统账号
- **步骤**:
  1. 在飞书群中 @机器人 发送任意消息
- **预期结果**:
  - 返回绑定引导卡片
  - 卡片包含「绑定账号」按钮 + 说明文字
  - 该消息 source 写入审计日志为 `"feishu_bot"`
- **测试方式**: 👤 人工在飞书客户端验证 + 审计日志后端验证

## 5. 可行性分析

| 检查项 | 结论 |
|--------|------|
| 是否需要模拟后端 API | Yes — 绑定 API (UI-161, UI-162 仅需 mock 后端绑定端点) |
| 是否依赖第三方服务 | **Yes** — 飞书 API（E2E 中 mock；人工测试使用真实飞书） |
| 是否需要特殊测试数据 | Yes — 绑定 Token mock、过期 Token mock |
| 是否需要新增 DB 集合 | No |
| 是否影响现有 API | No |
| 性能影响 | 无 |
| 实现复杂度 | Low — Web 绑定页仅 2 个 E2E 用例，标准表单验证 |
| 是否有无法实现的用例 | **UI-163 ~ UI-166: 全部标记为人工测试** — 飞书客户端侧功能无法在 Playwright 中自动化验证 |
