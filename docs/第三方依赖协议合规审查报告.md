## 二、高风险项（必须处理）

### 🔴 MinIO — AGPL v3（最终决策：已废弃，原因是维护状态而非协议）

| 项目 | 详情 |
|------|------|
| **协议** | GNU Affero General Public License v3.0 |
| **协议链接** | https://github.com/minio/minio/blob/master/LICENSE |
| **角色** | S3 兼容对象存储 |
| **风险 - 前期判断** | 最初认为 AGPL 传染风险导致必须替换 |
| **风险 - 严谨复审** | 经逐条对照 AGPL 触发条件，用户场景（独立进程 + 标准 HTTP/S3 API 调用 + 未修改源码 + 企业内网）完全符合"独立进程调用"例外，**实际不在 AGPL 传染范围内** |
| **最终决策** | **替换为 SeaweedFS（Apache 2.0）**—— 非因协议风险，而是 `minio-go` SDK 已停止维护，且 SeaweedFS 生态更活跃 |
| **行动**: ✅ **已决定替换为 SeaweedFS（Apache 2.0）**，轻量级 Go 实现，完全 S3 兼容 API，无任何开源协议风险。

**决策**: ✅ **选择 SeaweedFS（Apache 2.0）**。理由：1) Apache 2.0 无任何争议；2) minio-go SDK 已停止维护；3) SeaweedFS 纯 Go 实现，S3 兼容，Docker Compose 一键部署（`server -s3` 模式，端口 8333），社区活跃（30K+ GitHub stars）。

## 三、中风险项（需确认和处置）

### 🟡 1. Hasicorp Vault — BSL 1.1

| 项目 | 详情 |
|------|------|
| **协议** | Business Source License 1.1 (自 2023 年 8 月由 MPL 2.0 变更) |
| **角色** | 集中式敏感凭证管理 |
| **风险** | BSL 限制在生产环境中提供 Vault 作为**对外服务**（如提供 Vault-as-a-Service） |
| **内部使用** | ✅ 仅作为项目部署的**内部配套基础设施**，不对第三方提供 Vault 服务，BSL 通常可接受 |

**建议**:
- MVP 阶段可用 Vault 1.14.x（仍为 MPL 2.0，完全自由），锁定版本
- 或改用 **OpenBao**（Linux 基金会维护的 Vault 开源分支，MPL 2.0 协议，完全兼容）

**决策**: ✅ **保守方案 — 锁定 Vault 1.14.x（MPL 2.0）**。当前使用方式（内部密钥管理，不对外提供 Vault 服务）即使用最新版 1.18+ (BSL) 也合规，但固定版本可省去法务解释成本。

### 🟡 2. Redis — RSALv2 + SSPLv1（v7.4+）

| 项目 | 详情 |
|------|------|
| **协议** | Redis 7.4+ 采用 RSALv2 + SSPLv1 双协议（此前为 BSD） |
| **角色** | 缓存 + Redis Stream 消息队列 |
| **风险** | 使用 Redis 7.4+ 作为**对外提供的缓存服务**可能触发 SSPL 传染性条款 |

**建议**:
- Pin 到 **Redis 7.2.x**（BSD 协议，完全安全）
- 或改用 **Valkey 8.x**（Linux 基金会分支，BSD 协议，Redis 完全兼容的替代品，AWS/Google/Oracle 背书）

### 🟡 3. MongoDB — SSPL v1.0

| 项目 | 详情 |
|------|------|
| **协议** | Server Side Public License v1.0（非 OSI 批准） |
| **角色** | 统一业务数据存储 |
| **风险** | SSPL 的传染条款仅在提供 MongoDB 本身作为**数据库云服务**时触发 |

**分析**: 本项目**只把 MongoDB 作为后端存储使用**，不对外提供 MongoDB-as-a-Service。根据 MongoDB 官方 FAQ，此场景不受 SSPL 限制。绝大多数企业（包括阿里、腾讯、字节）都在此场景下使用 MongoDB Community Edition。

**建议**: 当前使用方式安全。如法务团队有顾虑，可选 **PostgreSQL + JSONB** 替代，但会增加迁移成本。

## 六、补充说明

- **copyleft 传染机制**: AGPL/SSPL 仅在你"分发"或"提供网络服务"给第三方时传染，纯内部使用不受影响
- **Go 标准库**: Go 使用 BSD-style license，所有标准库包（`database/sql`、`slog` 等）无商用风险
- **Python/R 运行时**: ~~Python (PSF License) 和 R (GPLv2)~~ — **已从方案中移除**。Worker 纯 Go 实现，不依赖外部脚本运行时
- **LLM API 调用**: OpenAI API 通过 HTTP 调用，不受模型本身的许可约束
